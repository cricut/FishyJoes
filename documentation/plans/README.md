# Plans

Implementation plans live here, separated by completion status.

## Status Rules

- `implemented/`: every step in the plan's definition of done has landed and
  been verified.
- `pending/`: future work, partially implemented work, or plans that have not
  yet satisfied their full definition of done.

When a pending plan is completed, move it to `implemented/` in the same commit
that records the final verification, or in a follow-up documentation-only
commit immediately after that implementation commit.
